# Changelog

## v1.5 — 2026-03-19 · 记忆树分层·上下文密度优化

### 变更
- 记忆树节点分三层：tier1骨架层(永远加载) / tier2场景层(按需) / tier3冷知识(手动)
- active节点从59个清理到33个，只保留原则和长期决策，操作记录/状态快照不进树
- 上下文生成三档：骨架版3KB / 按日期版tier1+tier2 / 完整版（有二次确认）
- unresolved问题独立为顶层open_issues字段，不再混入节点
- ai_note节点迁移至ai_observations.json，记忆树职责收窄
- 备忘录格式简化：`# @MEMO 需求：xxx | 注意：xxx`
- 选1/选3兼容新旧备忘录格式

### 优化
- 指令协议加定性说明：机器可解析协议，格式错误导致解析失败
- 静态文本压缩：中枢结构+开发规范10行→2行，操作速查6行→2行
- 修复骨架版对话摘要日期重复显示bug

### 架构说明
记忆树只存原则和长期决策，其他各归其位：未解决问题→open_issues，AI观察→ai_observations.json，操作记录→context_log。
